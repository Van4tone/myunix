进程标识符 ： 进程ID

		进程ID是唯一的，但是当一个进程终止后他的ID就可以再次重用。
		由于大多数UNIX操作系统是使用延时算法的，新建进程都是赋予不同于最近终止进程的ID.

		ID 为0 是调度进程，通常被称为交换进程，是内核的一部分。
		他并不执行任何磁盘上的程序，因此被称为系统进程。

		ID 为1 是init进程，在自举过程结束后由内核调用。
		他是一个普通的用户进程

		ID 为2 是守护进程，负责支持虚拟存储系统的分页操作。

fork 函数

	pid_t fork(void);
	一个现有的进程可以调用fork函数创建一个新进程。
	fork函数被调用一次，但是返回2次。
	子进程返回值为0，父进程返回值是子进程的进程ID.
	子进程是父进程的副本，子进程获得父进程的数据空间、堆和栈的副本。
	子进程只是拥有父进程的副本，并不是和父进程共享这些存储空间。
	父子进程共享正文段。
	写时复制
	父子进程执行的先后顺序并不确定，它依赖于内核所使用的调度算法。

vfork
    用于创建一个新进程，而该进程是 来exec一个新程序。
	vfork保证子进程先运行，他在调用exec或exit之后父进程才可能被调度运行。

wait waitpid 
	当一个进程正常或异常终止时，内核就向其父进程发送SIGCHLD信号。
	因为子进程终止是一个异步事件(这可以在父进程运行的任何时候发生），所以这种信号也是内核向父进程发的异步通知。
	父进程也可以选择忽略该信号，或者提供一个该信号发生时即被调用执行的函数（信号处理程序）。
	
	调用wait或者waitpid的进程可能会发生如下情况：
	*如果所子进程都还在运行，则阻塞。
	*如果一个子进程已终止，正等待父进程获取其终止状态，则取得该子进程的终止状态立即返回。
	*如果他没有任何子进程，则立即出错返回。

	如果进程由于接收到SIGCHILD信号而调用wait,则可期望wait立即返回。
	但是在任意时刻调用wait则进程可能会阻塞。

	pid_t wait(int *statloc);
	pid_t waitpid(pid_t pid, int *statloc, int options);
